Key Features

Natural Language Interface

Built-in agent-architect creates specialized agents from your descriptions. Building a fintech app? CAMI designs payment, security, and compliance agents for you.

Priority-Based Sources

Override default agents with your custom versions. When the same agent exists in multiple sources, lower priority numbers win. Customize without forking.

Three-Phase Methodology

Agent-architect embeds a three-phase methodology into every agent it creates: Research/Analyze (gather context and understand needs), Build/Core Action (execute the primary function), and Follow-up/Maintain (verify and refine). This framework ensures every CAMI agent is a complete specialist.

// Phases work as a cycle or independently

Research/Analyze Phase:
→ Gather relevant context
→ Assess current state and gaps
→ Understand requirements
→ Formulate informed approach

Build/Core Action Phase:
→ Apply domain expertise
→ Follow best practices
→ Make informed decisions
→ Produce high-quality outputs

Follow-up/Maintain Phase:
→ Verify quality standards
→ Document decisions
→ Identify remaining tasks
→ Guide future iteration

Modular by design: Agents run the full cycle for complex tasks, or jump to specific phases as needed. This embedded structure makes every CAMI agent systematic and complete.
